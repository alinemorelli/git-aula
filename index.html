Vamos lá!

Programar é como um divertido jogo de video-game, você começa no inicio, ultrapassa barreiras, vence etapas e, inevitavelmente, perde vidas. Mas nem sempre volta ao inicio após cada perda, existem etapas (checkpoints) que te dizem que até ali sua missão foi bem sucedida. No jogo esse checkpoint é uma cidade vencida no mapa a qual você pode sempre voltar, na programação, isso se chama "commit" e ele é possível graças ao controle de versão, ou seja, o Git!

O commit é importante por que a programação é feita de mais erros que acertos, ao longo dessa jornada você fará muitos testes e nem todos funcionarão como o esperado e não há nada de errado com isso. Para te auxiliar nesses testes existe também o Git e agora vamos ver um pouco de como ele funciona.

Vamos começar!

Após clicar no botão de download e efetuar a instalação (que consiste basicamente em apertar o "Next") você já terá o Git instalado em sua máquina e podemos dar inicio a diversão!

Vamos lá!

Abaixo podemos ver um terminal, mas não se assustem não é nada assim que vocês farão com eles, a não ser que queiram entrar na Matrix, mas não acho que seja o caso, voltando, este é um terminal e assim como o navegador compreende o "idioma" HTML o terminal compreende o idioma "Unix", ou seja, linhas de comando que dizem a ele o que fazer, neste caso, pegar as versões feitas por você de seu site e salvar de maneira segura no GitHub.

OLÁ TERMINAL EU QUERO FALAR COM VOCÊ!

Calma ai! As coisas não são tão simples assim, infelizmente, embora o terminal esteja 100% pronto para ser seu amigo ele ainda não faz a menor ideia do que você está falando, para isso você precisa falar a mesma língua que ele, então vamos por partes:

COISAS QUE O TERMINAL PODE FAZER:

  • Permitir que você navegue por ele entre pastas de seu computador;
  • Criar arquivos e pastas;
  • Indicar sua localização atual dentro das pastas;
  • Mostrar as pastas e arquivos dos seus diretórios (pastinhas criadas no Windows aquela "Nova Pasta")

COISAS QUE O TERMINAL NÃO PODE FAZER

  • Dar conselhos amorosos;
  • Programar páginas web sozinho;
  • Ensinar o caminho para o curso Eu Programo;
  • Te tirar da cama em dias frios;
  • Mapa astral.

MAS PRA QUE TUDO ISSO?

Saber o uso do terminal por linhas de comando é essencial para o desenvolvimento web, a primeira vista tudo isso pode parecer exagerado porque afinal de contas estamos apenas fazendo um pequeno site para o curso Eu Programo, mas as coisas não são bem assim, imagine-se em uma equipe multidisciplinar, ou onde seja necessário baixar os arquivos de seu projeto em uma máquina que não seja a mesma que você deu inicio a ele, ou ilustrando o caso aqui, que você tenha feito uma página muito legal durante o curso e queira dar continuidade a ela na sua casa, como você faria isso? Provavelmente se enviaria o arquivo por email, subiria ele na nuvem, ou até mesmo levaria ele em um pen drive, o terminal junto com o git automatizam esse processo e evitam duplicatas de arquivos, uma vez que ao invés de subir um novo arquivo toda vez que você faz uma atualização nele esse processo permite a atualização dele.

POR QUE?

O uso do git, como veremos no próximo capítulo dessa odisséia, é algo comum no universo profissional da programação, não se atentem a ideia de que se o projeto é pequeno não há necessidade de "tudo isso", lembre-se o projeto que hoje é pequeno amanhã pode ser um portal gigantesco de notícias, memorizem isso: "ORGANIZAÇÃO É IMPORTANTE" e ferramentas como o Git tornam ela mais fácil.

PRA QUEM?

Para toda e qualquer pessoa que queira desenvolver um projeto para web, neste caso, é sempre preciso pensar em processos, tudo na web são processos e etapas, é o tempo todo você falando com algo que não entende o seu idioma e nem a sua linha de raciocinio pictórica, ele é apenas linear e literal, preste atenção nisso!
